Offline Sync — Org Android & Windows
When Wi-Fi is available, the app automatically sends all stored progress to the server.
📶
Wi-Fi detected
App notices network is available
read local DB
📦
Collects unsynced data
Progress, quiz results, usage events
send batch
🖥️
Sync API receives data
Checks if this batch was already received
write to DB
✅
Progress saved in MongoDB
Visible in Admin Dashboard
confirm
🔄
Local data marked synced
Won't be sent again
Safe retry
Each sync batch has a unique ID. If the connection drops mid-sync and the app retries, the server recognises the duplicate and ignores it — so progress is never saved twice.
No Wi-Fi
Sync is skipped. Data stays safely in local storage and will sync next time the device connects.
